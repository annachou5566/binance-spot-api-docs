## 2025-12-17

> ⚠️ **Breaking change tại 2026-01-15 07:00 UTC:**
- Khi gọi các endpoint cần signature, phải **percent-encode payload TRƯỚC** khi tính signature. Nếu không sẽ nhận lỗi `-1022 INVALID_SIGNATURE`.

**REST API:** Cập nhật tài liệu ví dụ signed endpoint cho `POST /api/v3/order`.  
**WebSocket API:** Cập nhật tài liệu SIGNED request security.

---

## 2025-12-15

**Làm rõ về UTF-8 Encoding:**
- FIX, REST, WebSocket APIs: Nếu request chứa symbol có ký tự non-ASCII → response có thể chứa UTF-8.
- WebSocket Streams: `!miniTicker@arr` và rolling window streams có thể chứa UTF-8.
- User Data Stream: Có thể chứa UTF-8 nếu bạn sở hữu/giao dịch assets có tên non-ASCII.
- **Khuyến nghị:** Đảm bảo code xử lý được UTF-8.

---

## 2025-12-09

- Schema FIX SBE cập nhật `smallGroupSize16Encoding` trong `MarketDataSnapshot`.
- Cảnh báo mới: **Tránh dùng SQL keywords trong requests** (có thể bị WAF chặn).

## 2025-12-02 (Rollout ~2 tuần)

### General
- Parameter `symbol` và `symbols` hỗ trợ Unicode UTF-8.
- Xóa toàn bộ docs về `!ticker@arr` (đã deprecated từ 2025-11-14).

### FIX API
- Unicode UTF-8 được chấp nhận trong các tag: `Currency(15)`, `Symbol(55)`, `SecurityDesc(107)`, v.v.
- `NewOrderList <E>` chấp nhận `TriggerPriceDirection(1109)` mà không cần `TriggerPrice(1102)`.

### WebSocket Streams
- Hỗ trợ URL-encoded URLs.

> ⚠️ **Thay đổi tại 2025-12-18 07:00 UTC:**
- `ICEBERG_PARTS` tăng lên 25.
- FIX SBE support ra mắt.
- **OPO (One Pays the Other)** ra mắt:
  - REST: `POST /api/v3/orderList/opo`, `POST /api/v3/orderList/opoco`
  - WebSocket: `orderList.place.opo`, `orderList.place.opoco`
  - FIX: `NewOrderList <E>` thêm field `OPO (25046)`
- STP mode `TRANSFER` được thêm vào.
- **SBE schema 3:2** ra mắt; schema 3:1 deprecated (retire sau 6 tháng).
- FIX: `LastFragment (893)` deprecated.

## 2025-08-12

### WebSocket API
- Một WebSocket connection có thể subscribe nhiều **User Data Streams** cùng lúc.
- Thêm `userDataStream.subscribe.signature` (không cần login trước, hỗ trợ mọi API key type).
- Thêm `session.subscriptions` để liệt kê tất cả subscriptions đang active.
- `userDataStream.unsubscribe` hỗ trợ đóng nhiều subscriptions.

### SBE
- **Schema 3:1** ra mắt. Schema 3:0 deprecated (retire sau 6 tháng).
- Thêm các fields mới: `pegInstructionsAllowed`, `pricePeg`, `peggedPrice`, `subscriptionId`, v.v.

### FIX API
- FIX API không còn gửi `Reject <3>` khi client gửi reject message.
- Error messages rõ ràng hơn cho invalid/missing tags.

## 2025-04-07

### General
- FIX Market Data connection limits tăng từ 5 → **100** (từ January 2025, chưa được ghi nhận).
- Error code mới: `-2038` (order amend keep priority thất bại).
- `myTrades` weight: Nếu dùng `orderId` → weight = **5** (thay vì luôn là 20).
- Khi query/cancel order mà thiếu cả `orderId` lẫn `origClientOrderId` → lỗi `-1102` (thay vì `-1128`).

### FIX API (ra mắt 2025-04-21)
- FIX API kiểm tra `EncryptMethod(98)` = 0 tại Logon.
- FIX Order Entry giới hạn tối đa **10 kết nối đồng thời/account**.
- Connection rate limits được enforce:
  - FIX Order Entry: 15 lần/30 giây
  - FIX Drop Copy: 15 lần/30 giây
  - FIX Market Data: 300 lần/300 giây
- `News <B>` gửi countdown đến khi bị disconnect.
- `OrderCancelRequest <F>` và `<XCN>` cho phép cả `orderId` lẫn `clientOrderId`.

### User Data Streams
- **listenKey trên `wss://stream.binance.com` bị deprecated.**
- Chuyển sang dùng [User Data Stream qua WebSocket API](https://developers.binance.com/docs/binance-spot-api-docs/websocket-api/user-data-stream-requests).

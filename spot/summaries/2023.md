## 2023-12-04 (Rollout ~1 tuần)

### General
- Error message `Precision is over the maximum defined for this asset.` → `Parameter '%s' has too much precision.`
- `GET /api/v3/openOrderList` và `openOrderList.status` trả về kết quả theo thứ tự **ascending**.
- `GET /api/v3/allOrderList` và `allOrderLists` trả về đúng thứ tự ascending khi dùng `startTime`/`fromId`.
- Fix bug query order trả về lỗi `-2026 ORDER_ARCHIVED` sai.

### REST API
- Endpoint mới: `GET /api/v3/account/commission`
- Endpoint mới: `GET /api/v3/ticker/tradingDay`
- `GET /api/v3/avgPrice` response có thêm field `closeTime`.
- `GET /api/v3/klines` và `uiKlines` có thêm parameter `timeZone`.
- `POST /api/v3/order/test` và `POST /api/v3/sor/order/test` có thêm `computeCommissionRates`.

### WebSocket API
- Request mới: `account.commission`, `ticker.tradingDay`
- Session authentication (chỉ với Ed25519 keys):
  - `session.logon`, `session.logout`, `session.status`
- `avgPrice` có thêm `closeTime`.
- `klines` và `uiKlines` có thêm `timeZone`.
- Fix bug unsolicited pong gây disconnect.

### WebSocket Streams
- Stream mới: `<symbol>@avgPrice`
- `id` hỗ trợ 64-bit signed integer, alphanumeric strings (max 36 ký tự), `null`.

### User Data Stream
- `executionReport` với `x=TRADE_PREVENTION` → `l`, `L`, `Y` luôn là 0.
- Thêm fields mới: `pl` (prevented exec qty), `pL` (prevented exec price), `pY` (prevented exec notional).

## 2023-08-25 (Hiệu lực 2023-08-25 00:00 UTC)

**Rate limit thay đổi:**
- `CONNECTIONS` limit WebSocket API: **300 mỗi 5 phút**.
- `REQUEST_WEIGHT` (REST & WebSocket): **6,000 mỗi phút**.
- `RAW_REQUESTS` (REST): **61,000 mỗi 5 phút**.
- Chi phí kết nối WebSocket API: **2 weight** (trước là 1).

**Request weights tăng gấp đôi** cho nhiều endpoints. Xem chi tiết [CHANGELOG gốc](https://developers.binance.com/docs/binance-spot-api-docs/CHANGELOG#2023-08-25).

## 2023-03-13 (Rollout ~1 tuần)

### General
- Cải thiện error messages (xem bảng trong CHANGELOG gốc).
- Archived orders trả về error code `-2026` (thay vì `-2013`).
- `startTime` == `endTime` được chấp nhận (trước đây bị lỗi).
- Users vi phạm IP rate limit → bị ngắt kết nối WebSocket (status `418`).

> **~1 tuần sau:**
- `LOT_SIZE` và `MARKET_LOT_SIZE` filter: Thay đổi cách tính `stepSize`.
- Reverse `MARKET` orders không đủ fill → status `EXPIRED` (không còn luôn là `FILLED`).

### REST API
- `DELETE /api/v3/order` và `POST /api/v3/order/cancelReplace` thêm `cancelRestrictions`.

## 2023-01-19

**Tính năng mới: Self-Trade Prevention (STP)**

- Ngăn orders match với orders từ cùng account hoặc cùng `tradeGroupId`.
- Order status mới: `EXPIRED_IN_MATCH`.
- REST mới: `GET /api/v3/myPreventedMatches`.
- Params mới: `selfTradePreventionMode` cho `POST /api/v3/order`, `POST /api/v3/order/oco`, v.v.
- WebSocket mới: `myPreventedMatches`.
- User Data Stream: Execution type mới `TRADE_PREVENTION`.
